# 4. 前端 — 计费、模板、文件与 Webhook 集成

对应后端需求：[`../../docs/requirements/04-终端用户-计费模板文件与集成.md`](../../docs/requirements/04-终端用户-计费模板文件与集成.md)。

## 4.1 计费（Billing）

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-BIL-01 | **计费记录** | 表格 + 时间范围 + 分页；导出（若产品需要，以后端为准） | 租户隔离；大表性能可接受 |
| FE-BIL-02 | **追加记录**（若对终端开放） | 表单提交；幂等提示（若后端返回） | 成功反馈明确 |
| FE-BIL-03 | **预算** | 列表与 upsert 表单；展示剩余额度或告警状态 | 保存成功刷新 |
| FE-BIL-04 | **计费设置** | 分区块表单；PATCH 后提示生效范围 | 与 `model-router` 说明文案一致（产品提供） |
| FE-BIL-05 | **模型路由解析** | 调试或配置预览：输入上下文 → 展示选用模型 | 与设置一致 |
| FE-BIL-06 | **额度检查** | 高消费操作前可调用并展示「允许/拒绝」原因 | 超预算时用户可理解下一步（充值/调整预算） |

## 4.2 模板与 Agent 商城

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-TM-01 | **模板浏览** | 列表/卡片 + 详情页 + 预览（`preview`） | 加载失败可重试 |
| FE-TM-02 | **模板导入** | 确认后长请求（约 60s）；进度或异步通知 | 失败信息可行动 |
| FE-MK-01 | **商城列表/详情** | 商品信息、价格或积分（以实现为准） | — |
| FE-MK-02 | **购买/安装** | 下单流程；成功跳转已安装 Agent 或刷新列表 | 计费失败明确提示 |

## 4.3 文件（Files）

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-FILE-01 | **上传** | `multipart` 上传；进度条；大小/类型超限前置校验（与后端一致） | 成功后展示文件 ID 或链接 |
| FE-FILE-02 | **文件列表与操作** | 列表、预览链接、下载、删除；预签名 URL 过期前刷新或提示 | 删除后列表移除 |

## 4.4 Webhook 集成

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-WH-01 | **Webhook 配置 CRUD** | 表单含 URL、事件类型、secret（创建时仅一次完整展示）；列表不展示明文 secret | 403 无管理权限时隐藏入口 |
| FE-WH-02 | **投递历史** | 表格 + 状态 + 重试信息（若返回） | 可筛选时间 |
