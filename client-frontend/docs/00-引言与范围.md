# 0. 引言与范围（前端）

## 0.1 文档目的

将 Foundry **网关对外能力**转化为**可验收的前端需求**：用户在哪些页面完成哪些任务、界面应呈现何种状态（加载/成功/空/错误）、与 JWT 及租户上下文如何配合。本文档不替代设计稿，但为信息架构、组件行为与测试用例提供统一语言。

## 0.2 技术栈与仓库约定（当前）

| 项 | 约定 |
|----|------|
| 运行时 | 浏览器（现代 Chromium / Safari / Firefox 最近两个大版本） |
| 框架 | React + Vite（见 `client-frontend`） |
| 路由 | 客户端路由；公开页与受保护路由分离（如 `ProtectedRoute` / `PublicOnlyRoute`） |
| HTTP | 经配置的 **API 基址**指向 **Gateway**（典型全局前缀 `/api`） |
| 实时 | Socket.IO 客户端连接网关 **`/collaboration`** 命名空间（需有效 JWT，见协作文档） |

具体环境变量（如 `VITE_API_BASE_URL`）以实现为准，部署文档需写明。

## 0.3 系统边界（前端可见）

| 边界 | 说明 |
|------|------|
| **认证** | 注册、登录、刷新、登出、OAuth 回调页；令牌存取策略（内存 + `sessionStorage`/`localStorage` 等）需与安全评审一致 |
| **业务 API** | `GET/POST/PATCH/DELETE` 至 `/api/v1/*`；请求头携带 `Authorization: Bearer <accessToken>`（及网关要求的其它头） |
| **管理 API** | `/api/admin/*` 仅管理员角色；前端应对 403 给出明确提示并可引导联系管理员 |
| **Webhook 接收** | 通常为服务端入口；若控制台配置「测试投递」，UI 仅展示结果而非直连 `receive` 端点 |
| **指标** | `/api/metrics` 一般由运维抓取；管理控制台可选嵌入只读面板或外链 |

## 0.4 角色与路由策略（简化）

| 角色 | 前端表现 |
|------|----------|
| **访客** | 仅可访问公开路由（登录、注册、OAuth 回调等）；访问受保护 URL 时重定向至登录页并保留 `returnUrl` |
| **登录用户** | 可访问租户内业务模块；无权限操作按钮置灰或隐藏（以后端 403 为准） |
| **管理员** | 额外展示管理入口（动态路由、API Key、IP 规则、审计等），具体以产品与路由守卫为准 |

## 0.5 多租户与公司上下文

- 业务数据隔离由后端与 JWT 上下文保证；前端应**显式展示当前公司/租户名称**（若产品提供切换器，切换后需刷新依赖上下文的列表与详情）。
- **验收关注点**：切换上下文后，不应残留上一上下文的缓存数据（或需失效查询缓存）。

## 0.6 非目标

- 本文档不规定像素级视觉规范（色板、字体由设计系统另文定义）。
- 不列举每个表单字段；字段级校验与文案以后端错误码及 Swagger 为准。
