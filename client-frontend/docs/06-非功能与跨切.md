# 6. 前端 — 非功能与跨切关注点

对应后端 [`../../docs/requirements/06-后台异步与能力边界.md`](../../docs/requirements/06-后台异步与能力边界.md) 中的系统边界；本节描述**客户端**应满足的横向要求。

## 6.1 错误、空态与加载

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| FE-NF-01 | 全局 **HTTP 错误映射**：网络失败、4xx/5xx、超时 | 用户可理解文案；可重试场景提供按钮 |
| FE-NF-02 | **空列表/无权限** 区分展示 | 403 与「暂无数据」文案不同 |
| FE-NF-03 | 列表与详情 **skeleton 或占位**，避免布局跳动 | 长列表虚拟滚动（若数据量大，产品要求时） |

## 6.2 安全（前端）

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| FE-NF-04 | **令牌存储** 方案经评审；避免在 URL 中传递长期令牌 | OAuth 回调处理完毕即清理 query |
| FE-NF-05 | **XSS**：对用户生成内容做转义或安全渲染策略 | 富文本场景以产品安全规范为准 |
| FE-NF-06 | **CSRF**：若后端要求 CSRF token，前端在变更类请求中携带 | 与网关中间件配置一致 |

## 6.3 国际化与可访问性

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| FE-NF-07 | 文案抽离，便于 **i18n**（若产品多语言） | 日期/数字格式随 locale |
| FE-NF-08 | 关键交互可 **键盘** 完成；焦点可见；图标按钮有 `aria-label` | 与 WCAG 2.1 AA 目标对齐（按产品等级） |

## 6.4 性能与离线

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| FE-NF-09 | 路由级 **code split**；首屏资源体积可控 | Lighthouse 性能预算（由团队定义） |
| FE-NF-10 | **离线**仅提示网络不可用（Foundry 默认非离线优先应用） | 不静默失败 |

## 6.5 与 Worker / 日志服务的关系

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| FE-NF-11 | 终端用户**不直接调用** Worker HTTP；UI 不暴露内部 Worker 地址 | — |
| FE-NF-12 | 若集成 `apps/logging` 批量上报，前端 SDK **采样/批处理** 避免阻塞主线程 | 失败不阻断用户操作 |

## 6.6 文档与实现同步

- 路由与菜单变更时更新本系列对应章节。
- Swagger 仅在非生产开启时，前端开发环境以 **mock 或直连开发网关** 为主，不依赖生产 Swagger。
