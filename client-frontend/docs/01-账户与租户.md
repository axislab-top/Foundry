# 1. 前端 — 账户与租户

对应后端需求：[`../../docs/requirements/01-终端用户-账户与租户.md`](../../docs/requirements/01-终端用户-账户与租户.md)。

## 1.1 注册与登录

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-ACC-01 | **注册页** | 表单含邮箱/用户名/密码等（以后端为准）；提交中按钮 loading；成功跳转登录或自动登录（产品定） | 409/校验错误在字段旁或顶部展示可读文案；不泄露用户是否存在（若后端统一模糊提示则前端一致） |
| FE-ACC-02 | **登录页** | 凭证登录；失败展示通用或字段级错误；触发限流时展示 429 友好提示 | 多次失败后用户能理解「稍后再试」 |
| FE-ACC-03 | **会话续期** | 访问受保护页时若 access token 将过期，静默刷新或引导重新登录 | 刷新失败时清除本地会话并跳转登录，不无限重试 |
| FE-ACC-04 | **登出** | 全局入口触发 `logout` API 后清除本地令牌并跳转公开页 | 登出后按返回键不应恢复受保护内容（依赖路由与缓存清理） |
| FE-ACC-05 | **微信 OAuth** | 「使用微信登录」跳转授权 URL；**回调页**解析 token 或错误码并完成会话建立 | 回调页有明确成功/失败态；错误可重试回登录页 |

## 1.2 用户资料与列表（管理员/授权用户）

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-USR-01 | **用户列表** | 表格或卡片 + 分页 + 搜索/筛选（若 API 支持）；加载骨架或 spinner | 空列表有引导；仅本租户数据 |
| FE-USR-02 | **用户详情/编辑** | 查看与编辑分步或同页；PATCH 成功后乐观更新或重新拉取 | 403 时整页或区块级提示 |
| FE-USR-03 | **管理员创建用户** | 独立表单或抽屉；创建成功可跳转详情或留在列表并高亮新行 | 权限不足不展示创建入口 |

## 1.3 OAuth 绑定（第三方账号）

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-OAUTH-01 | **绑定第三方** | 在「账户设置」展示已绑定列表；提供绑定入口（跳转或弹窗） | 仅本人或管理员可见（与后端一致） |
| FE-OAUTH-02 | **解绑**（若后端支持） | 二次确认后调用 API | 成功后列表即时更新 |

## 1.4 租户上下文（横切）

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| FE-TEN-01 | 顶栏或侧栏展示**当前租户/公司**；可选切换器 | 切换后路由与数据一致；避免闪烁可保留 layout 仅刷新内容区 |
| FE-TEN-02 | 所有 `/api/v1` 请求自动附带认证头；401 统一处理 | 全局 401 拦截：清会话、跳转登录、可带 `returnUrl` |
