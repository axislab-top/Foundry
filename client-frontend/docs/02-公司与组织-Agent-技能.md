# 2. 前端 — 公司、组织、Agent 与技能

对应后端需求：[`../../docs/requirements/02-终端用户-公司与组织-Agent-技能.md`](../../docs/requirements/02-终端用户-公司与组织-Agent-技能.md)。

## 2.1 公司（Companies）

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-CO-01 | **公司列表/详情** | 列表分页；详情页展示基础信息与状态标签 | 筛选与排序与 API 一致；无数据空态 |
| FE-CO-02 | **创建/编辑公司** | 表单校验；创建成功进入详情或返回列表 | 冲突与校验错误可理解 |
| FE-CO-03 | **状态变更** | 启用/停用等操作用确认对话框；成功后刷新详情 | 非法状态迁移时展示后端错误信息 |

## 2.2 组织结构（Organization）

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-ORG-01 | **组织树** | 树形控件展示 `GET .../tree`；支持展开/折叠；可选懒加载子节点（若后端支持） | 大型树滚动与性能可接受 |
| FE-ORG-02 | **节点增删改** | 侧栏或弹窗表单；删除前确认 | 父子约束错误时提示明确 |
| FE-ORG-03 | **移动节点** | 拖拽或「移动到…」选择父节点；禁止成环时按钮禁用或提交后展示错误 | 与 `organization.node.move` 行为一致 |
| FE-ORG-04 | **节点下 Agent / 汇报链** | 子页面或抽屉展示列表与链视图 | 与组织数据一致 |
| FE-ORG-05 | **节点与技能绑定** | 多选技能列表 + 绑定/解绑；展示当前绑定 | 绑定后列表即时反映 |
| FE-ORG-06 | **组织审计**（若产品需要） | 表格 + 筛选 + 分页 | 与 `organization.audit.logs` 字段对齐 |
| FE-ORG-07 | **知识摘要** | 触发请求后长 loading（约 30s 超时配置）；超时展示重试 | 用户知悉「耗时较长」 |

## 2.3 Agent

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-AG-01 | **Agent 列表/详情/表单** | CRUD 流程完整；详情含状态、所属节点等 | 租户隔离；删除二次确认 |
| FE-AG-02 | **批量招募** | 批量提交进度或异步结果展示；部分失败时列出失败项 | 长超时 UI 不阻塞整页（可后台轮询或通知） |
| FE-AG-03 | **生效技能、记忆统计、审计** | 详情页 Tab 或区块；审计表格分页 | 空数据有说明 |
| FE-AG-04 | **审批、状态、分配节点** | 操作按钮与当前状态联动；非法迁移禁用或报错 | 与后端状态机一致 |
| FE-AG-05 | **技能绑定** | 与组织侧类似的绑定 UI；解绑确认 | 列表与详情同步 |

## 2.4 技能（Skills）

| 需求 ID | 界面/场景 | 交互与状态 | 验收要点 |
|---------|-----------|------------|----------|
| FE-SK-01 | **技能库** | 列表 + 创建/编辑/删除；被引用时删除约束以后端为准 | 删除失败说明原因（如仍被引用） |
