# 02 — 用户需求与系统能力映射

本节将「用户/客户可能提出的需求」映射到 **Foundry 已具备或部分具备的后端能力**。粒度为**产品/交付**级别；具体路由与表结构见 `docs/features/*` 与迁移文件。

---

## 1. 需求分类总览

| 需求大类 | 用户故事（示例） | 主要落点（代码/服务） | 成熟度说明 |
|----------|------------------|----------------------|------------|
| 账户与访问 | 注册、登录、令牌刷新、管理员管用户 | Gateway 认证 + API Users/Auth/OAuth | **高**：网关与 API 模块完整度高 |
| 多租户与企业 | 多公司隔离、企业资料与状态 | Tenant + RLS、`CompaniesModule` | **高**：迁移与守卫体系存在 |
| 组织与数字员工 | 组织树、节点、Agent、技能绑定与审批 | `OrganizationModule`、`AgentsModule`、`SkillsModule` | **中高**：控制面与事件较全，执行与工具链需环境 |
| 实时协作 | 房间、消息、成员、运营台类场景 | `CollaborationModule` + Gateway WebSocket | **中高**：后端与契约文档存在，前端需对接 |
| 企业记忆 / RAG | 文档摄入、检索、摘要、与任务联动 | `MemoryModule` + Worker ingest | **中**：异步与 RPC 路径具备，向量/模型依赖配置 |
| 任务与自动化 | 拆解、仪表盘、长任务、心跳 | `TasksModule` + Worker 多监听器 | **中**：事件与调度存在，自治深度见差距文档 |
| 用量与计费 | 消耗记录、预算、模型路由 | `BillingModule` + Worker 计费监听 | **中**：入账与信号链路有，商业化规则需验收 |
| 模板与扩展 | 模板预览/导入、物化 | `TemplatesModule` + `template-imported` 监听 | **中**：幂等与导入路径有，一键全量物化需逐项验收 |
| 文件与对象存储 | 上传下载、预签名 URL | `FilesModule` + 多存储适配器 | **高**：MinIO/S3/OSS/本地 |
| 集成与出站 | Webhook 配置、接收、重试 | `apps/webhooks` | **高**：见 `implemented-features` Webhooks 章节 |
| 运维与治理 | 动态路由、API Key、IP 策略、审计、指标 | Gateway 管理模块与中间件 | **中高**：功能存在，生产策略需配置验证 |

---

## 2. 按典型「用户角色」的需求

### 2.1 平台管理员 / 运维

- **动态路由与灰度**：动态路由表 + 刷新缓存、静态回退（Gateway）。
- **密钥与访问控制**：API Key CRUD 与轮换、IP 黑白名单。
- **审计与合规**：审计日志查询（含公司维度扩展设计）。
- **可观测性**：健康检查、`/api/metrics`（Prometheus 文本）、日志拦截与链路相关模块。

### 2.2 企业租户管理员

- **公司与组织**：公司生命周期、组织树与节点变更、审计类能力（以模块与迁移为准）。
- **Agent 与技能**：Agent CRUD、与组织节点联动、技能平台/组织级管理（执行日志等表迁移已存在）。
- **成员与协作**：协作房间/消息/成员；实时通道依赖 Gateway WebSocket 约定。

### 2.3 业务用户 / 协作者

- **登录与账号**：网关注册/登录/刷新、微信 OAuth 路径（配置依赖环境）。
- **协作沟通**：房间与消息 API；实时推送依赖 Socket 连接与 JWT。
- **任务与待办**：任务 OS 相关 RPC 与 Worker 侧拆解/完成/心跳事件。

### 2.4 集成开发者

- **出站 Webhook**：事件订阅、签名、历史与重试。
- **入站 API**：经 Gateway 统一鉴权与路由；部分路由可走 RPC 白名单治理。

---

## 3. 与「AI 公司 / 自治运行」叙事的对齐

若产品叙事包含 **「组织可视化 → Agent 配置 → 实时协作 → 自治运行 → 记忆沉淀 → 成本控制」**，当前后端提供的**可验证支点**包括：

- 组织 + Agent + 技能 **控制面** 与 **领域事件**；
- 协作 **HTTP + WebSocket** 双通道；
- 记忆 **摄入与检索 RPC** + Worker 异步摄入；
- 任务 **拆解/心跳/完成** 与计费 **消耗/预算信号** 监听；
- Worker 侧 **LangGraph / 自治编排** 相关模块（具体拓扑与层级监管见 [`03-honest-boundaries.md`](./03-honest-boundaries.md) 与根目录差距章节）。

---

## 4. 本映射不承诺的内容

- **前端 UI**、移动端 App：不在默认 `apps` 交付范围内。
- **单键完整商业闭环**：如自动对账、发票、支付网关，需另行集成。
- **未经配置即用的 AI 效果**：模型密钥、向量库、配额与成本控制依赖部署环境。
