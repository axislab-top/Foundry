## 2. A. 数据层隔离（TenantModule + RLS）

### 2.1 核心表 `company_id` / `tenant_id`

| 结论 | 说明 |
|------|------|
| **大部分通过** | 租户业务表在迁移中广泛包含 `company_id`，并与 `companies(id)` 关联；协作、记忆、任务、审批、计费等迁移中均有体现（例如 `1767865000000_AddTenantFoundationAndRls.ts`、`1767878000000_AddMemoryModuleTables.ts`、`1767879000000_AddTasksModuleTables.ts`、`1767875000000_AddCollaborationTables.ts`、`1770700000000_M4ApprovalRequestsAndKillSwitch.ts` 等）。 |
| **设计例外：`users`** | `infrastructure/postgres/migrations/1767855312397_CreateUsersTable.ts` 中 `users` **无** `company_id`；多公司通过 `company_memberships` 等表关联。若清单字面要求「users 表也有 company_id」，则与当前模型不一致——应视为**平台账户表**，隔离验收应落在 membership + RLS 业务表上。 |

### 2.2 PostgreSQL RLS + FORCE ROW LEVEL SECURITY

| 结论 | 说明 |
|------|------|
| **核心租户表：已通过迁移配置** | 多处迁移含 `ENABLE ROW LEVEL SECURITY`、`FORCE ROW LEVEL SECURITY`，策略使用 `current_setting('app.current_tenant', true)::uuid`（与 `docs/TenantModule验收说明.md` 描述一致）。 |
| **持续风险** | **新增表**若未跟迁移加 RLS，会造成缺口；仓库内**无**自动化「全表 RLS 覆盖率」校验脚本（本次检索未发现）。 |

### 2.3 TenantContext 与 `app.current_tenant`

| 结论 | 说明 |
|------|------|
| **API/TypeORM 路径：已实现** | `infrastructure/tenant/src/services/tenant-typeorm-context-bootstrapper.service.ts` 在 `QueryRunner.connect` 后调用 `SQL_SET_SESSION_CURRENT_TENANT` 并传入 `tenantContext.getCompanyId()`；若 `companyId` 为空则跳过设置。 |
| **Worker** | `apps/worker/src/app.module.ts` 导入 `TenantModule`；与 `docs/TenantModule验收说明.md` 中「监听器 + runWithCompanyId」描述一致（本次未逐文件审计每个消费者）。 |

### 2.4 Redis / MinIO 前缀

| 结论 | 说明 |
|------|------|
| **Redis：部分按文档** | `docs/TenantModule验收说明.md` 写明业务缓存 `company:${companyId}:...`，平台级 nonce/api_key 等为全局前缀。若清单要求「所有 Redis key」带公司前缀，则**与文档声明的平台键策略不一致**（属设计取舍，非疏漏）。 |
| **MinIO：Service 层已强制写入前缀（部分修复）** | **P1**：`StorageService` + `storage-tenant-path.util` 对所有写操作解析为 `companies/{companyId}/...`；legacy `memory/{companyId}/` **写**被拒绝。适配器仍接收**已解析**的 key 并做纵深断言（`companies/`、`memory/` 读、`skills/`、`platform/` 等）。**残留风险**：须保证**所有**业务仅通过 `StorageService` / 已校验 RPC 访问存储，绕过则仍有风险。 |

### 2.5 绕过 tenant 过滤

| 结论 | 说明 |
|------|------|
| **未发现典型「全局 SQL」模式** | 未做全量人工审计；建议对 `createQueryBuilder` 无 `company_id` 条件的查询做专项 review。 |
| **Webhooks 服务** | `apps/webhooks/src/app.module.ts` **未**导入 `TenantModule`，仅 `UserContextMiddleware` + `GuardsModule`。若 Webhooks 写租户数据，需单独证明上下文注入方式；否则应在文档中明确 **Webhooks 仅平台级、不触租户库**。 |
