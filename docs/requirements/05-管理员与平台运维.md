# 5. 管理员与平台运维

以下接口位于 **Gateway**，前缀一般为 **`/api`**，需 **JWT + 管理员角色**（以各 Controller 守卫为准）。

## 5.1 动态路由治理

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| ADM-RT-01 | 管理员能 **增删改查** 网关动态路由规则。 | `POST/GET/PUT/DELETE .../admin/routes`（含 `:id`） | 错误配置不导致网关不可用；变更后行为可预期。 |
| ADM-RT-02 | 管理员能**刷新**动态路由缓存，使规则立即生效。 | `POST /api/admin/routes/refresh` | 刷新后新规则被匹配。 |

## 5.2 API Key

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| ADM-KEY-01 | 管理员能管理 **API Key**（创建、列表、撤销等）。 | `/api/admin/api-keys` CRUD | 密钥仅创建时完整展示；轮换后旧密钥失效。 |
| ADM-KEY-02 | 管理员能对指定 Key 执行**轮换**。 | `POST .../api-keys/:id/rotate` | 旧 Key 立即或按策略失效。 |

## 5.3 IP 访问控制

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| ADM-IP-01 | 管理员能维护 **白名单/黑名单** IP 规则。 | `POST/DELETE/GET .../admin/ip-filter/whitelist|blacklist` | 与 `IpFilterMiddleware` 联动；规则持久化。 |

## 5.4 审计

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| ADM-AUD-01 | 管理员能**查询审计日志**（过滤 + 分页）。 | `GET /api/admin/audit-logs` | 含公司维度扩展字段（以实体为准）；敏感字段脱敏。 |

## 5.5 可观测性

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| OPS-MET-01 | 运维能拉取 **Prometheus 指标**文本。 | `GET /api/metrics` | 格式符合 Prometheus exposition。 |
| OPS-HL-01 | 运维能对各服务做**健康检查**（直连各服务或经网关策略）。 | 各服务 `GET .../health`（见 `implemented-features.md`） | 数据库与依赖状态可见。 |

## 5.6 安全能力（条件生效）

网关挂载 **签名、防重放、CSRF、IP 过滤** 等中间件；对未携带约定 Header 的请求常**自动跳过**，以便兼容纯 JWT 客户端。集成方若启用强校验，需同时配置 Header 与密钥策略（详见 `architecture/overview.md` 与网关中间件代码）。

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| SEC-01 | 可选 **HMAC 签名**请求。 | 带签名时非法签名被拒绝。 |
| SEC-02 | 可选 **时间戳 + nonce 防重放**。 | 重放请求被拒绝。 |
| SEC-03 | 可选 **CSRF**（如携带 `x-csrf-token` 时校验）。 | 浏览器场景防护有效。 |
