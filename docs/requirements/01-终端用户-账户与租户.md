# 1. 终端用户 — 账户与租户

## 1.1 用户注册与登录（网关）

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| ACC-01 | 作为访客，我希望用邮箱等凭证**注册**账号，以便进入系统。 | `POST /api/auth/register`（网关 `AuthController`），成功后返回访问令牌与刷新令牌。 | 重复邮箱/用户名返回冲突；成功返回 201 及令牌。 |
| ACC-02 | 作为访客，我希望**登录**并获取会话令牌。 | `POST /api/auth/login`；带 **限流**（如每 IP 时间窗口内次数上限）。 | 错误凭证 401；频繁失败触发 429。 |
| ACC-03 | 作为登录用户，我希望能**刷新**访问令牌而无需重新输入密码。 | `POST /api/auth/refresh`。 | 有效刷新令牌换发新访问/刷新令牌；无效则拒绝。 |
| ACC-04 | 作为登录用户，我希望能**登出**（使会话失效或废弃刷新令牌，具体以后端策略为准）。 | `POST /api/auth/logout`（需 JWT）。 | 未带令牌拒绝；成功后客户端清除本地令牌。 |
| ACC-05 | 作为用户，我希望能使用**微信 OAuth** 登录或绑定（若环境已配置）。 | `GET /api/auth/wechat/authorize`、`GET /api/auth/wechat/callback`。 | 回调带回可建立会话的令牌或错误信息。 |

## 1.2 用户资料与生命周期（API，经网关 `/api/v1/users`）

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| USR-01 | 作为访客，我希望能**自助注册**（部分部署可能同时暴露 `POST /api/users/register` 于 API 直连场景）。 | API `UsersController` 注册端点（`@Public`）。 | 与网关注册二选一或并存时行为一致、无重复账号。 |
| USR-02 | 作为管理员，我希望能**创建**用户。 | `POST /api/v1/users`（RPC `users.create`），需管理员角色。 | 非管理员拒绝。 |
| USR-03 | 作为授权用户，我希望能**分页查询**用户列表（含筛选/搜索等）。 | `GET /api/v1/users`（`users.findAll`）。 | 仅返回当前租户可见数据。 |
| USR-04 | 作为授权用户，我希望能**查看/更新**指定用户。 | `GET`/`PATCH /api/v1/users/:id`。 | 越权访问被拒绝。 |
| USR-05 | 作为管理员，我希望能**删除**用户。 | `DELETE /api/v1/users/:id`（`users.remove`）。 | 管理员权限；删除语义与软删以 API 为准。 |

## 1.3 认证校验（供网关或内部）

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| AUTH-VAL-01 | 作为系统组件，我需要**校验凭证**以完成登录流程。 | `POST /api/auth/validate` 或 RPC `auth.validate`（见路由表）。 | 合法凭证与非法凭证响应区分明确。 |

## 1.4 OAuth 第三方账号（API `/api/v1/oauth/...`）

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| OAUTH-01 | 作为登录用户，我希望能将**第三方账号绑定**到指定用户。 | `POST /api/v1/oauth/bind/:userId`。 | 需有效 JWT 与权限。 |
| OAUTH-02 | 作为登录用户，我能查看已绑定的**第三方账号列表**。 | `GET /api/v1/oauth/accounts/:userId`。 | 仅本人或管理员可见（以实现为准）。 |
| OAUTH-03 | 作为登录回调链路，系统需能**查找或创建**用户（公开端点）。 | `POST /api/v1/oauth/find-or-create`（路由配置 `authRequired: false`）。 | 仅用于受控回调场景，勿滥用。 |

## 1.5 租户上下文（横切需求）

| 需求 ID | 描述 | 验收要点 |
|---------|------|----------|
| TEN-01 | 请求携带的公司/用户上下文由网关与 API 中间件解析并注入（如 `UserContextMiddleware`）。 | 业务接口行为与头部/令牌一致；缺失上下文时错误可预期。 |
