# 3. 终端用户 — 协作、记忆、任务与仪表盘

默认前缀 **`/api/v1`**，需 **JWT**。

## 3.1 协作（Collaboration）

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| COL-01 | 我能查看**房间列表**、**主房间**及单个房间详情。 | `GET .../collaboration/rooms`、`.../main`、`.../rooms/:roomId` | 仅本租户可见房间。 |
| COL-02 | 我能**发送消息**并**按房间拉取消息列表**。 | `POST /v1/collaboration/messages`；`GET .../rooms/:roomId/messages` | 顺序与分页；权限不足拒绝。 |
| COL-03 | 我能**搜索消息**（关键词或条件以实现为准）。 | `GET /v1/collaboration/messages/search` | 性能与超时（配置约 15s）。 |
| COL-04 | 我能查看房间**成员**并**添加/移除成员**。 | `GET .../members`；`POST .../members`、`.../members/remove` | 角色与房间权限。 |
| COL-05 | 我能请求**房间摘要**（可能调用模型，耗时）。 | `POST /v1/collaboration/rooms/summary` | 异步或同步语义以实现为准。 |
| COL-06 | 我能从**组织节点批量加入成员**到协作上下文。 | `POST .../members/add-from-organization-node` | 与组织数据一致。 |
| COL-07 | 我能创建**部门房间**或同类房间。 | `POST /v1/collaboration/rooms/department` | 创建后可在列表中查到。 |
| COL-08 | 我能按条件**搜索组织节点**（协作侧）。 | `GET /v1/collaboration/organization-nodes/search` | 与组织模块边界清晰。 |

### 3.1.1 实时通道（WebSocket）

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| COL-WS-01 | 作为登录用户，我希望能通过 **WebSocket** 接收房间/协作相关实时通知。 | 网关 Socket.IO 命名空间 `/collaboration`，JWT 鉴权；详见 `docs/collaboration-websocket-contract.md` 与 `02-gateway-access-and-realtime.md`。 | 连接需有效令牌；断线重连策略由客户端配合。 |

## 3.2 记忆与知识（Memory / RAG）

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| MEM-01 | 我能**写入**记忆条目。 | `POST /v1/memory/entries` → `memory.entries.store` | 租户隔离；大负载超时（约 30s）。 |
| MEM-02 | 我能对记忆库做**语义检索**。 | `POST /v1/memory/search` | 返回相关性排序可接受。 |
| MEM-03 | 我能请求**摘要**类能力。 | `POST /v1/memory/summarize`（超时约 60s） | 空库或错误输入可预期。 |
| MEM-04 | 我能**导入文档**到记忆管道（同步/异步）。 | `POST .../documents/ingest`、`.../ingest-async` | 大文件走异步；任务状态可查（若实现暴露）。 |

## 3.3 任务（Tasks）

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| TASK-01 | 我能**创建/列表/查看/更新/删除**任务。 | `/v1/tasks` CRUD；`GET .../tasks/:id` | 状态机与租户隔离。 |
| TASK-02 | 我能请求将任务**拆解**为子任务（可能长耗时）。 | `POST /v1/tasks/breakdown`（超时约 60s） | 失败重试与部分成功策略。 |
| TASK-03 | 我能查看任务的**树结构**与**执行日志**。 | `GET .../tasks/:id/tree`；`GET/POST .../execution-logs` | 日志追加顺序正确。 |
| TASK-04 | 我能**分配任务**与**更新进度**。 | `POST .../assign`；`PATCH .../progress` | 指派人权限校验。 |

## 3.4 仪表盘（Dashboard）

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| DASH-01 | 我能查看**公司级摘要**仪表盘数据。 | `GET /v1/dashboard` → `dashboard.companySummary` | 与任务/Agent 等数据大致一致。 |
| DASH-02 | 我能查看**计费相关摘要**（与账单模块联动）。 | `GET /v1/dashboard/billing` → `dashboard.billingSummary` | 与计费记录可对账。 |
