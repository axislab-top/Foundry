# 4. 终端用户 — 计费、模板、文件与 Webhook 集成

## 4.1 计费（Billing）

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| BIL-01 | 我能**查看计费记录**列表。 | `GET /v1/billing/records` → `billing.records.list` | 时间范围与分页；租户隔离。 |
| BIL-02 | 系统或集成方能**追加**一条计费记录（如模型调用）。 | `POST /v1/billing/records` → `billing.record.append` | 幂等与必填字段。 |
| BIL-03 | 我能查看并**创建/更新预算**（upsert）。 | `GET /v1/billing/budgets`；`PUT /v1/billing/budgets` | 预算超限策略与提醒（以实现为准）。 |
| BIL-04 | 我能查看并**更新计费设置**（模型路由、限额等，以实现为准）。 | `GET`/`PATCH /v1/billing/settings` | 配置生效后影响 `model-router` 与 `check-allowance`。 |
| BIL-05 | 我能请求**模型路由解析**（给定上下文返回选用模型）。 | `POST /v1/billing/model-router/resolve` | 与设置一致。 |
| BIL-06 | 我能**检查额度/是否允许**继续消费。 | `POST /v1/billing/check-allowance` | 超预算时返回明确可处理结果。 |

## 4.2 模板与 Agent 商城（Templates / Marketplace）

| 需求 ID | 用户故事 | API 形态 | 验收要点 |
|---------|----------|----------|----------|
| TM-01 | 我能**浏览模板列表**与**查看详情/预览**。 | `GET /v1/templates`；`GET .../templates/:id`；`GET .../:id/preview` | 平台级或租户级可见性以实现为准。 |
| TM-02 | 我能将模板**导入**到公司环境（可能长耗时）。 | `POST /v1/templates/:id/import`（超时约 60s） | 失败回滚或错误信息清晰。 |
| MK-01 | 我能浏览 **Agent 商城**并查看商品详情。 | `GET /v1/marketplace/agents`、`GET .../marketplace/agents/:id` | 与已安装 Agent 不冲突。 |
| MK-02 | 我能**购买/安装**商城 Agent（业务规则以实现为准）。 | `POST .../marketplace/agents/:id/purchase` | 计费或权限联动。 |

## 4.3 文件（Files）

文件上传/下载多为 **二进制或 multipart**，网关路由中 **`/v1/files` 部分为 RPC**（列表、URL、信息、删除），**上传**可走捕获规则 **`/v1/*`** 转发至 API 的 `/api/*`（见 `routes.config.ts` 中 `/v1/*` → `target: '/api'`）。

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| FILE-01 | 我能**上传**文件到对象存储（MinIO/S3/OSS/本地等由配置决定）。 | API `POST /api/files`（multipart） | 大小与类型限制；返回可访问路径或 ID。 |
| FILE-02 | 我能**列出、获取 URL、查看元信息、下载、删除**文件。 | RPC：`files.list`、`files.getUrl`、`files.getInfo`；HTTP 下载见 `implemented-features.md` | 预签名 URL 过期时间；删除后不可访问。 |

## 4.4 Webhook 集成（配置面 + 接收面）

| 需求 ID | 用户故事 | 实现要点 | 验收要点 |
|---------|----------|----------|----------|
| WH-01 | 作为租户管理员，我能**CRUD Webhook 配置**并查看**投递历史**。 | 经网关 `GET/POST/PATCH/DELETE /api/v1/webhooks` 等（RPC 至 webhooks 服务）；`GET .../webhooks/:id/history` | 仅授权用户；secret 不明文返回。 |
| WH-02 | 作为外部系统，我能向平台**投递 Webhook**（异步接受）。 | `POST /api/webhooks/receive`（通常无 JWT）；头 `x-webhook-event` 必填；可选签名校验 | 立即 **202**；后续转发与重试见 webhooks 服务实现。 |
